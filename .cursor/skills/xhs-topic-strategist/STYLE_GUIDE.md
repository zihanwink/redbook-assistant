# 小红书风格规范

## 目标

内容应：**真实、具体、小切口、有场景、有经验感、可收藏、可互动**，不像 AI、不像论文、不过度标题党。

## 禁用表达

删除或改写：

- 本文将 / 综上所述 / 首先其次最后 / 总的来说
- 近年来 / 随着时代发展
- 大家可以根据自身情况
- **围绕「X」从 Y 角度切入**
- **关键词｜入门指南** 式机械拼接
- 作为一个 AI

## 推荐表达

- 场景开头：「如果你也是通勤党…」
- 反常识：「我以前以为…后来发现…」
- 踩坑复盘：「这类情况先别急着…」
- 清单式：「我会先看这 3 个点」
- 谨慎表述：「参考」「思路」「适合收藏对照」

## 标题规则（18 字内优先）

必须有：**小切口 + 对象/人群 + 痛点或反差 + 收藏价值**

可用模板：

1. 给【人群】的【问题】避坑清单
2. 【场景】别急着【动作】，先看这几点
3. 我以前以为【误区】，后来发现【认知】
4. 【人群】做【事】前，先问自己这 5 个问题
5. 【数字/预算】实测｜【结果/路线】

禁止：绝对化承诺、伪造排名、 guaranteed 效果。

## 描述规则（50–80 字）

结构：**目标用户 → 具体痛点 → 内容切入点 → 读者收获**

口语自然，每条描述必须与标题角度不同。

## 笔记结构（扩展场景）

1. 场景钩子 → 2. 痛点共鸣 → 3. 误区/冲突 → 4. 具体方法 → 5. 清单/示例 → 6. 温和提醒 → 7. 评论互动
